> **For agentic workers:** REQUIRED SUB-SKILL: Use superpowers:subagent-driven-development (recommended) or superpowers:executing-plans to implement this plan task-by-task. Steps use checkbox (`- [ ]`) syntax for tracking.

**Goal:** Reduce route_task p99 latency to under 5ms by optimizing SQLite PRAGMAs, caching prepared statements and agent stats, adding composite indices, and reducing hot-path allocations.

**Architecture:** All changes are in arbiter-mcp (DB layer, agents cache) and arbiter-core (lazy path building). No API changes. Pure performance optimization with backward-compatible behavior.

**Tech Stack:** Rust, rusqlite (prepare_cached), std::collections::HashMap for cache

### Task 1: SQLite PRAGMAs + Composite Index

**Files:**
- Modify: `arbiter-mcp/src/db.rs:120-140` (open method)
- Modify: `arbiter-mcp/src/db.rs:622-702` (SCHEMA_V1)

This task adds 3 performance PRAGMAs to the `open()` method and a composite index to the schema.

- [ ] **Step 1: Add PRAGMAs to `Database::open()`**

In `arbiter-mcp/src/db.rs`, in the `open()` method, after the existing `foreign_keys` pragma (line 136), add:

```rust
        // Performance PRAGMAs (safe with WAL mode).
        conn.pragma_update(None, "synchronous", "NORMAL")
            .context("Failed to set synchronous=NORMAL")?;

        conn.pragma_update(None, "cache_size", -8000)  // 8MB cache
            .context("Failed to set cache_size")?;

        conn.pragma_update(None, "mmap_size", 67108864)  // 64MB mmap
            .context("Failed to set mmap_size")?;
```

Note: `synchronous=NORMAL` is safe with WAL — data won't corrupt on crash, though the last few transactions might be lost (acceptable for a routing engine). `cache_size=-8000` means 8MB (negative = KiB). `mmap_size` allows memory-mapped I/O for reads.

- [ ] **Step 2: Add composite index to schema**

In `arbiter-mcp/src/db.rs`, at the end of `SCHEMA_V1` (before the closing `";`), add:

```sql
CREATE INDEX IF NOT EXISTS idx_agent_stats_agent_type ON agent_stats(agent_id, task_type);
```

This index covers the `get_agent_stats` and `get_agent_stats_by_category` queries that filter by `agent_id`.

- [ ] **Step 3: Run tests**

Run: `cargo test -p arbiter-mcp`
Expected: ALL PASS (PRAGMAs don't affect in-memory DB behavior, index is additive)

- [ ] **Step 4: Commit**


### Task 3: In-Memory Agent Stats Cache

**Files:**
- Modify: `arbiter-mcp/src/agents.rs` — add cache layer over DB queries

**Context:** Each `route_task` call queries `get_agent_stats`, `get_running_tasks`, and `get_recent_failures` for every candidate agent (typically 3). That's 9 DB queries per request. An in-memory cache with a short TTL (e.g., 1 second) eliminates most of these queries while staying fresh enough for routing decisions.

- [ ] **Step 1: Add cache infrastructure to `AgentRegistry`**

In `arbiter-mcp/src/agents.rs`, add:

```rust
use std::time::{Duration, Instant};

/// Cache entry for agent info with TTL.
struct CachedAgentInfo {
    info: AgentInfo,
    cached_at: Instant,
}

/// How long cached agent info remains valid.
const CACHE_TTL: Duration = Duration::from_secs(1);
```

Change `AgentRegistry` to include a cache:

```rust
use std::cell::RefCell;

pub struct AgentRegistry<'a> {
    db: &'a Database,
    configs: HashMap<String, AgentConfig>,
    cache: RefCell<HashMap<String, CachedAgentInfo>>,
}
```

Use `RefCell` because `AgentRegistry` methods take `&self` (not `&mut self`), and the cache is an internal optimization that doesn't change the public API.

- [ ] **Step 2: Add cache logic to `get_agent_info`**

```rust
pub fn get_agent_info(&self, agent_id: &str) -> Result<Option<AgentInfo>> {
    // Check cache first
    {
        let cache = self.cache.borrow();
        if let Some(cached) = cache.get(agent_id) {
            if cached.cached_at.elapsed() < CACHE_TTL {
                return Ok(Some(cached.info.clone()));
            }
        }
    }

    let config = match self.configs.get(agent_id) {
        Some(c) => c.clone(),
        None => return Ok(None),
    };

    let running_tasks = self.db.get_running_tasks(agent_id)?;
    let stats = self.db.get_agent_stats(agent_id)?;
    let recent_failures = self.db.get_recent_failures(agent_id, 24)?;

    let success_rate = if stats.total_tasks > 0 {
        Some(stats.success_rate)
    } else {
        None
    };
    let avg_duration = if stats.total_tasks > 0 {
        Some(stats.avg_duration_min)
    } else {
        None
    };
    let avg_cost = if stats.total_tasks > 0 {
        Some(stats.avg_cost_usd)
    } else {
        None
    };

    let info = AgentInfo {
        agent_id: agent_id.to_string(),
        config,
        running_tasks,
        success_rate,
        avg_duration_min: avg_duration,
        avg_cost_usd: avg_cost,
        recent_failures,
    };

    // Update cache
    self.cache.borrow_mut().insert(
        agent_id.to_string(),
        CachedAgentInfo {
            info: info.clone(),
            cached_at: Instant::now(),
        },
    );

    Ok(Some(info))
}
```

- [ ] **Step 3: Add `invalidate_cache` method**

- [ ] **Step 5: Add cache tests**

```rust
#[test]
fn cache_hit_avoids_db_query() {
    let (db, agents) = setup();
    let registry = AgentRegistry::new(&db, &agents).unwrap();

    // First call populates cache
    let info1 = registry.get_agent_info("claude_code").unwrap().unwrap();

    // Change DB state
    db.increment_running_tasks("claude_code").unwrap();

    // Second call within TTL should return cached (stale) data
    let info2 = registry.get_agent_info("claude_code").unwrap().unwrap();
    assert_eq!(info1.running_tasks, info2.running_tasks, "cache should return stale data within TTL");
}

#[test]
fn invalidate_cache_forces_refresh() {
    let (db, agents) = setup();
    let registry = AgentRegistry::new(&db, &agents).unwrap();

    // Populate cache
    let info1 = registry.get_agent_info("claude_code").unwrap().unwrap();
    assert_eq!(info1.running_tasks, 0);

    // Change DB state and invalidate
    db.increment_running_tasks("claude_code").unwrap();
    registry.invalidate_cache("claude_code");

    // Should get fresh data
    let info2 = registry.get_agent_info("claude_code").unwrap().unwrap();
    assert_eq!(info2.running_tasks, 1);
}
```

### Task 4: Lazy Decision Path Building

**Files:**
- Modify: `arbiter-core/src/policy/decision_tree.rs:147-200`

**Context:** Currently, `predict()` builds a `Vec<String>` path by calling `format!()` at every tree node. For a depth-7 tree, that's ~7 `format!` allocations per prediction. Since the path is only used for audit logging (not for the routing decision itself), we can build it lazily — collect raw data during traversal, format strings only at the end.

- [ ] **Step 1: Collect raw path data instead of formatted strings**

Replace the path building in `predict()`. Instead of pushing formatted strings at each node, collect a lightweight enum:

```rust
/// Raw decision path entry (allocation-free during traversal).
enum PathEntry {
    Node {
        node_idx: usize,
        feat_idx: usize,
        feat_val: f64,
        threshold: f64,
        went_left: bool,
    },
    Leaf {
        class_name_idx: usize,
        confidence: f64,
    },
}
```

Define this inside the `predict()` method (or as a private type in the module). Then at the end of predict(), convert to strings:

```rust
let path: Vec<String> = raw_path
    .iter()
    .map(|entry| match entry {
        PathEntry::Node { node_idx, feat_idx, feat_val, threshold, went_left } => {
            let feat_name = self
                .feature_names
                .get(*feat_idx)
                .cloned()
                .unwrap_or_else(|| format!("feature[{feat_idx}]"));
            let dir = if *went_left { "left" } else { "right" };
            let cmp = if *went_left { "<=" } else { ">" };
            format!("node {node_idx}: {feat_name} ({feat_val:.2}) {cmp} {threshold:.2} -> {dir}")
        }
        PathEntry::Leaf { class_name_idx, confidence } => {
            let class_name = self
                .class_names
                .get(*class_name_idx)
                .cloned()
                .unwrap_or_else(|| format!("class_{class_name_idx}"));
            format!("leaf: {class_name} (confidence={confidence:.3})")
        }
    })
    .collect();
```

- [ ] **Step 2: Update the traversal loop**

Replace the current path.push(format!(...)) calls with raw_path.push(PathEntry::...) calls.

In the leaf branch:
```rust
raw_path.push(PathEntry::Leaf {
    class_name_idx: class,
    confidence,
});
```

In the left/right branches:
```rust
raw_path.push(PathEntry::Node {
    node_idx,
    feat_idx,
    feat_val,
    threshold: node.threshold,
    went_left: true, // or false for right
});
```
